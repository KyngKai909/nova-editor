# Getting started

This walks you through your first edit in about five minutes.

## 1. Open the dashboard

Go to the app and click **Start building** (or open `/dashboard`). The dashboard
is your home base — every project you import shows up here as a card with a live
preview. Projects are stored in your browser (and, optionally, on your disk).

No account is required to start.

## 2. Open a project

Click **New project** and pick one:

- **GitHub** — paste a public repo URL, or connect GitHub to browse and import
  your own (including private) repos.
- **Folder** — choose a folder on your computer (Chromium browsers).
- **Paste** — drop in a single HTML or JSX/TSX component.
- **Try the sample** — the fastest way to look around.

See [Importing a project](./importing.md) for details.

## 3. Make a visual edit

Once the project opens, you're in the **editor**:

- **Click any element** on the canvas to select it.
- Use the **inspector** on the right to change layout, spacing, typography, and
  color. Changes apply live and are written straight into your source.
- **Double-click text** to edit it in place.
- Switch **breakpoints** (desktop / tablet / mobile) in the top bar to design
  responsively.

None of this spends an AI token — it's direct manipulation of your code.

## 4. (Optional) Ask the AI

Click the **AI** button in the top bar to open the assistant. Add your own API
key (any provider), then ask it to build or change something. It reads and edits
your real files, and the canvas updates as it works. See
[The AI assistant](./ai-assistant.md).

## 5. Ship it

Click **Publish** to review a clean diff of exactly what changed. From there you
can download the files, save to your folder, or — if the project is connected to
GitHub — **commit, push, or open a pull request**. See
[Publishing & GitHub](./publishing.md).
